## 一、数据模型（树状导航，非搜索引擎）

```
/schools                              ← 20所高校列表 + 评分
  │
  └─ /schools/{URL编码学校名}         ← 院系列表 + 评分
```

## 五、爬虫设计方案

### 5.1 搜索匹配策略

网站**没有搜索API**，采用 **URL构造+模糊匹配** 策略：

```
用户输入: "孙立峰" + "清华大学" + "计算机系"
  │
  ├─ 预加载 /schools → 获取所有学校URL
  │     匹配: "清华大学" → /schools/%E6%B8%85%E5%8D%8E%E5%A4%A7%E5%AD%A6
  │
  ├─ 加载 /schools/清华大学 → 获取院系列表
  │     匹配: "计算机" → /schools/清华大学/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%A7%91%E5%AD%A6
  │
  └─ 加载 /schools/清华大学/计算机系 → 获取导师列表
        匹配: "孙立峰" → /teacher/98800 → 详情页
```

### 5.2 搜索引擎模式（推荐）

预构建一个**本地倒排索引**（启动时一次性完成）：
1. 遍历所有学校 → 院系 → 导师
2. 建立映射: `{导师名 → {teacher_id, score, school, dept}}`
3. 搜索时 O(1) 查找
4. 总请求量: ~20学校 × ~30院系 ≈ 600次请求

### 5.3 返回数据格式

```python
{
    "name": "孙立峰",
    "university": "清华大学",
    "department": "计算机科学与技术系",
    "overall_score": 5.0,
    "review_count": 2,
    "reviews": [
        {
            "author": "AI总结",
            "content": "非常nice的老师，每周一对一指导，实验室经费充足...",
            "source": "daoshipingjia.net",
            "source_url": "https://daoshipingjia.net/teacher/98800"
        }
    ],
    "source": "daoshipingjia.net",
    "detail_url": "https://daoshipingjia.net/teacher/98800"
}
```

### 推荐方案

**方案A: 按需查找**（轻量，每次搜索时爬取）
- 每次搜索：学校匹配(1次) → 院系匹配(1次) → 导师列表匹配(1次) → 详情(1次)
- 4次HTTP请求/次搜索
- 适合低频使用

**方案B: 预建索引**（推荐，启动时完成）  
- 启动时全量爬取 → 构建本地倒排索引
- ~600次请求（~10分钟）
- 搜索时O(1)查找，0网络延迟
