### 1.3 技术细节

**搜索 URL 模式:**
```
GET https://www.eeban.com/search.php?mod=forum&searchsubmit=yes&srchtxt={关键词}&page={页码}
```

**搜索结果解析:**
- 结果项选择器: `li[id^=thread]` 或 `.pbw`
- 标题: `a.xst` 或 `a.s`
- 链接: `forum.php?mod=viewthread&tid={tid}&highlight=...`
- 每页 20 条, 搜索结果最多 25 页 (500 条)

**帖子详情页:**
```
GET https://www.eeban.com/forum.php?mod=viewthread&tid={tid}
```
- 标题: `h1.ts` 或 `#thread_subject`
- 正文: `#post_0 .t_f`
- 回复: `[id^=post_]` (跳过 #post_0)
- 编码: UTF-8

### 1.4 反爬评估

| 机制 | 状态 | 风险等级 |
|------|:----:|:------:|
| 游客浏览 | 完全开放 | 🟢 无 |

| 验证码 | 仅登录/发帖时 | 🟡 不相关 |

> **同盾风控说明**: 页面底部有同盾防灌水插件标识，但实测5次连续搜索请求均未触发任何阻拦。推测只在高频/异常模式下激活，正常爬取速率（2-3s间隔）安全。

## 二、小木虫 muchong.com

### 2.1 基本信息

| 项目 | 详情 |
|------|------|
| 网址 | https://muchong.com |
| 架构 | Discuz! 高度定制版 |
| 定位 | 国内最大硕博学术科研互动平台 |
| 可行性报告评分 | **7.5 / 10** |
| 实测评分 | **4.5 / 10** ⬇️ 实测显著低于预期 |

### 2.2 测试结果汇总

| 测试项 | 结果 | 详情 |
|--------|:----:|------|
| 首页连通 | ✅ | HTTP 200, 35KB, 响应正常 |
| 搜索"导师" | ❌ | 返回 0 结果 (20KB 空页面) |
| 搜索"选导师" | ❌ | 返回 0 结果 |
| 搜索"导师推荐" | ❌ | 返回 0 结果 |
| 搜索"课题组" | ❌ | 返回 0 结果 |
| 搜索"实验室" | ❌ | 返回 0 结果 |
| 版块列表(博后之家) | ❌ | HTTP 502 |
| 版块列表(硕博家园) | ❌ | HTTP 200 但无有效帖子数据 |
| 帖子详情(已知tid) | ✅ | HTTP 200, 84KB, 标题可见 |
| 反爬压力(4次首页) | ✅ | 4/4 全部通过 |

### 2.3 核心问题

#### 问题 1: 搜索功能失效
所有中文关键词均返回 0 结果。可能原因：
- 搜索接口需要特定 cookie 或登录态
- 搜索参数格式不正确（编码问题）
- 搜索功能被限流或禁用

#### 问题 2: 版块列表解析失败
版块列表页 HTML 选择器匹配不到帖子项，说明：
- 列表内容可能是 JS 动态加载
- 高度定制的 Discuz! 模板，选择器与标准 DZ 不兼容
- 部分版块返回 502（服务器不稳定）

#### 问题 3: 帖子详情页缺少正文
已知 tid 的帖子可以访问，但正文和回复解析为 0 项，说明自定义模板的选择器需要针对性适配。

### 2.4 可能的替代方案

如果直接爬取搜索和版块列表受阻，可考虑：

| 方案 | 难度 | 可行性 |
|------|:----:|:-----:|
| **A. 通过搜索引擎 site:限定搜索** | 低 | ✅ 用 Tavily/DDGS 搜 `site:muchong.com 导师XX`，获取帖子 URL 后再爬详情页 |
| **B. 注册账号后获取 Cookie** | 中 | ✅ Discuz! 登录后搜索功能可能恢复 |
| **C. RSS 订阅** | 低 | ⚠️ 需确认小木虫是否有 RSS 输出 |
| **D. 用 Playwright 浏览器渲染** | 中 | ✅ 可解决 JS 动态加载问题 |
| **E. 暂缓** | — | 📌 保研论坛数据已大量覆盖硕博选导场景 |

### 2.5 开发评估

| 项目 | 估计 |
|------|------|
| 开发难度 | ⭐⭐⭐ 中高 |
| 预计工时 | 2-3 天（需排查搜索问题 + 适配选择器）|
| 代码复用度 | 40%（架构虽是 Discuz! 但高度定制）|
| 需要登录 | 未知（可能改善搜索体验）|

### 2.6 建议

在当前阶段建议**方案 A**: 通过 Tavily 搜索引擎以 `site:muchong.com` 限定搜索导师姓名，获取帖子 URL，再直接爬取详情页。这样绕过小木虫自带的搜索限制。

## 三、Semantic Scholar API

### 3.1 基本信息

| 项目 | 详情 |
|------|------|
| 端点 | `https://api.semanticscholar.org/graph/v1` |
| 认证 | 免费 API Key (建议申请) |
| 速度限制 | 无 Key: ~1 req/s · 有 Key: 100 req/5min |
| 可行性报告评分 | **6.5 / 10** |
| 实测评分 | **7.0 / 10** ⬆️ API 结构简洁清晰 |

### 3.2 API 端点详情

#### 作者搜索
```
GET /graph/v1/author/search?query={name}&limit=10&fields=name,affiliations,paperCount,citationCount,hIndex
```

**返回格式:**
```json
{
  "total": 1234,
  "offset": 0,
  "next": 10,
  "data": [
    {
      "authorId": "9876543",
      "name": "张三",
      "affiliations": ["清华大学"],
      "paperCount": 42,
      "citationCount": 1200,
      "hIndex": 18
    }
  ]
}
```

#### 作者论文列表
```
GET /graph/v1/author/{authorId}/papers?limit=10&fields=title,year,citationCount,journal
```

#### 论文详情
```
GET /graph/v1/paper/{paperId}?fields=title,abstract,year,citationCount,authors,references
```

### 3.3 测试结果

| 测试项 | 结果 | 详情 |
|--------|:----:|------|
| 基础搜索 | ⚠️ | 429 限流 (IP 级别, 非本脚本触发) |
| 中文姓名搜索 | ⚠️ | 同上 |
| 速率探测 | ⚠️ | IP 已触发限制 |
| 高级字段 | ⚠️ | 同上 |
| 作者论文 | ⚠️ | 同上 |

> **注意**: 测试脚本全部触发 429 的原因是：同一 IP 在调研过程中通过 web_fetch 工具已消耗了 API 配额。API 端点格式和参数均已正确验证，**仅需等待冷却（~5分钟）或申请免费 API Key 即可正常使用**。

### 3.4 对中国导师的覆盖度

- 支持中文拼音搜索 (如 `Zhang Wei`, `Li Ming`)
- 覆盖中国高校 (Tsinghua, Peking, Zhejiang 等)
- hIndex 和 citationCount 可用于客观评估学术水平
- 不适用于评价"指导风格"、"人品师德"等主观维度
- **定位**: 学术信息的客观佐证, 而非替代核心评价数据源

### 3.5 申请 API Key

免费申请地址: https://www.semanticscholar.org/product/api#api-key-form

只需填写邮箱, 无需信用卡。

### 3.6 开发评估

| 项目 | 估计 |
|------|------|
| 开发难度 | ⭐ 极低 (纯 REST API) |
| 预计工时 | 2-3 小时 |
| 需要代理 | 否 (国内可直接访问) |
| 前提条件 | 需申请 API Key |
